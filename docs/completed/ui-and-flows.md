## 2. 主要フロー

### 2.1 イベント前の準備フロー（管理者）

1. `/admin` にアクセス
2. イベント名を入力して保存
3. 登壇者ごとに以下を登録
   - 名前
   - LTタイトル
   - スライドURL（Google Drive PDF の direct URL）
4. 登壇者一覧で順番をドラッグ＆ドロップで調整
5. 必要であれば「ランダム並び替え」で順番をシャッフル

### 2.2 イベント当日の進行フロー（管理者）

1. 会場スクリーンのブラウザで `/screen?layout=top` などを開く
2. 管理用PCで `/admin` を開く
3. 「次へ」ボタンで現在の登壇者を切り替える
4. 各LT開始時にタイマーをスタート
5. LT終了後にタイマーリセット → 次の登壇者へ

### 2.3 登壇者によるスライド操作フロー

1. 登壇者は、自分の LT に紐づいたURL `/remote/[talkId]` を開く
   - 事前共有 or QRコードでの案内
2. 画面には
   - 自分の名前
   - LTタイトル
   - 「前のスライド」「次のスライド」ボタン
3. 「次のスライド」をタップすると、`slide_states.current_page` が更新される
4. `/screen` 側で Realtime 購読しているスライドページが更新され、表示が切り替わる

### 2.4 参加者によるコメント投稿フロー

1. 会場スクリーンや司会が、QRコードで `/comment/[talkId]` へのアクセスを案内
2. 参加者はスマホでアクセスし、
   - 表示名（任意）
   - コメント本文
     を入力して送信
3. コメントは `comments` テーブルに紐付いて保存される
4. `/screen` 側は `comments` を Realtime 購読しており、
   - 現在登壇者の `talk_id` に紐づくコメントだけをタイムラインに表示
   - 新規コメントがINSERTされるたびに画面に即時反映される

## 3. Tailwind 前提のUIメモ

- 全体背景は黒 or ダークグレー（プロジェクタで映えるように）
- テキストは白ベース＋アクセントカラー（例: `emerald-400`）
- フォントサイズ
  - イベント名・登壇者名・タイトルは `text-3xl〜4xl` 以上
  - コメントタイムラインは `text-lg` 程度
- レイアウト切り替え
  - `flex-col` / `flex-row` 切り替え
  - `layout` クエリに応じてクラスを切り替える実装イメージ
